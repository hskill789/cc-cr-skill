---

第一行：版本/MR 信息行，格式（用 `##` 标题，飞书渲染为二级标题）：
- tag 模式：`## Release <old_tag> → <new_tag>: <描述> — Code Review`
- mr 模式：`## MR !<iid>: <title> — Code Review`

第二行：作者/状态/分支，格式：
- tag 模式：`提交者: <committer列表> | 变更: <file_count> 个文件，+N / -N 行 | Commit: N 个`
- mr 模式：`作者: <author_name> | 状态: <state> | 分支: <source_branch> → <target_branch>`

第三行：
- tag 模式：`变更: <file_count> 个文件，+N / -N 行 | Commit: N 个`（若已在第二行合并则省略）
- mr 模式：`变更: <file_count> 个文件，+N / -N 行 | Commit: N 个`

---

**概览**：2-3 句话概括做了什么，涉及哪些模块。时间信息：tag 模式写 tag 创建时间（北京时间）；MR 模式写 created_at 创建时间和 merged_at 合并时间（均转换为北京时间）。末尾加 GitLab 链接：
- tag 模式：`Tags：[查看所有版本](GITLAB_LINK)`
- mr 模式：`MR 列表：[查看所有 Release MR](GITLAB_LINK)`
